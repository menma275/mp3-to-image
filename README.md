# mp3-to-image // Glitch Format Converter

MP3ファイルを画像（バイナリマップ）に変換し、ファイル形式変換の過程で生じる非意図的なグリッチ（バイナリノイズ）を可視化・可聴化するWebアプリケーション。

## デプロイ先
https://mp3-to-image.vercel.app/

---

## 主な機能

### 1. MP3 ➔ IMAGE（画像化とグリッチ）
* **ピクセルレンダリング**: MP3のバイト列を直接ピクセル（RGB 24bit, RGBA 32bit, Grayscale 8bit, RGB16 16bit）として解釈して描画する。
* **アスペクト比調整**: 指定したアスペクト比（1:1, 16:9, 4:3, 3:2, 21:9）に合わせて、総ピクセル数から歪みのない最適な縦横サイズを自動計算する。
* **グリッチパイプライン**:
  * **データ破壊率 (Corrupt)**: バイト列をランダムに破壊・置換する。
  * **バイトソート強度 (Sort)**: データブロックを整列させ、擦れたようなグリッチラインを形成する。
  * **XORビット演算**: 特定のマスク値でビット反転を施す。
* **カラーマッピング**: 5種の配色プリセット（Raw, Glitch Gold, Cyberpunk, High Contrast Mono, Organic Haze）を適用する。

### 2. 統合インタラクティブ・タイムライン
* **波形描画**: MP3の波形データを解析し、タイムライン背景に音量レベルを描画する。デコード失敗時はブロックノイズ波形を自動生成する。
* **範囲スライス**: タイムライン上のハンドルを左右にドラッグ（またはタッチ）し、画像化・再生する開始/終了バイト範囲をトリミングする。
* **シーク再生**:
  * 指定したスライス範囲内のみを再生する。
  * タイムライン上をクリックまたはドラッグすることで、再生位置をリアルタイムにシークする。
  * デコード再生（MP3として再生）とRAW再生（生バイトをPCM音響ノイズとして再生）に対応する。

### 3. IMAGE ➔ MP3（画像の音響化）
* **ソニフィケーション**: 画像の画素ストリーム（RGBA / RGB / 輝度）を音声データとして解釈し、指定したサンプリングレートで再生する。
* **偽装エクスポート**: RAWノイズ再生のほか、ダミーのMP3フレームヘッダーの挿入、またはWAVコンテナヘッダーを埋め込んだ偽装音声ファイルを保存する。

## 技術スタック

* **コア**: HTML5 / JavaScript (ES6 modules)
* **描画**: HTML5 Canvas (低レイヤ・ピクセルデータ操作)
* **音響**: Web Audio API (動的デコード、スライス再生、PCMシンセサイズ)
* **ビルドツール**: Vite
* **パッケージマネージャー**: pnpm

## 開発環境のセットアップ

### 1. 依存関係のインストール
```bash
pnpm install
```

### 2. 開発サーバーの起動
```bash
pnpm run dev
```

### 3. プロダクションビルド
```bash
pnpm run build
```
